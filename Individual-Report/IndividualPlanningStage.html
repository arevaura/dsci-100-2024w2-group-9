The players.csv dataset contains 7 different variables with 196 observations, meaning there are 1372 values in total. The data variables mostly surround the user's account information, each observation is an individual player.

  1. Qualitative variable named experience, indicates how familiar the user is with Minecraft. The users were asked to choose the description that matched them the most, the categories include:
    • Beginner
    • Amateur
    • Regular
    • Pro
    • Veteran.
  2. Qualitative variable named subscribe, indicates the user's subscription status being either true or false. This variable is automatically true when the user signs up.
  3. Qualitative variable named hashedEmail, shows the custom token assigned to the user. The users will receive an email with their custom generated token once they attach their email address.
  4. Quantitative variable named hours_played, represents the number of hours the user has played on the server for. This variable will be continuously tracked across the player's time spent on the server.
  5. Qualitative variable named name, contains the user's name used when registering.
  6. Qualitative variable named gender, where the user inputs their gender.
  7. Quantitative variable named Age, represents the user's age given by the user. An issue with this dataset is that there is one user whose name was written as NA.

The sessions.csv dataset contains 5 different variables with 1535 observations, resulting in 7675 values in total. The data observations each represent one playing session on the server by a user, and all the variables except the first record significant timestamps of each session.

  1. Qualitative variable named hashedEmail, shows the which shows the custom token assigned to the user. The users will receive an email with their custom generated token once they attach their email address.
  2. Quantitative variable named start_time, indicates the exact date and time the user's session began. This data is extracted from the server catalog.
  3. Quantitative variable named end_time, indicates the exact date and time the user's session ended. This data is extracted from the server catalog.
  4. Quantitative variable named original_start_time, indicates the time the user's session began in unix time stamp. This data is recorded by calculations done with start_time.
  5. Quantitative variable named original_end_time, indicates the time the user's session ended in unix time stamp. This data is recorded by calculations done with end_time. An error within the last two variables is that using unix time stamp is not precise enough to actually provide helpful data in terms of time, as the values are rounded to too little decimals, causing there to be no observable differences between original_start_time and original_end_time.

Can the experience level and age of a user predict the number of hours the user spent on the server? Different levels of experience may lead to different interest levels for the game, and age might also be a limiting factor depending on how occupied the age group is on average.
