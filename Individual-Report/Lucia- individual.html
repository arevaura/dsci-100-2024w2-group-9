Data Description

The dataset consists of two CSV files:

  1. players.csv: Contains a list of all unique players, including data about each player.
  2. sessions.csv: Contains a list of individual play sessions by each player, including data about the session.

Players Data (players.csv)

  • Number of Observations: 196 players
  • Number of Variables: 7 primary variables
  • Description of Variables:
Variable Name Type Description

subscribe Boolean Whether the player subscribed to the game (TRUE/FALSE)

played_hours Numeric (float) Total hours played on the server

original_start_time Numeric (milliseconds) UNIX timestamp representing the session start time

Data Collection & Potential Issues:

  • If a player disconnects unexpectedly or quits before a session is logged properly, the session duration might be recorded incorrectly or omitted.
  • played_hours or session_duration could have missing values for players who joined but never actively played.
  • If most players have low playtime, predicting high-activity users might be difficult due to class imbalance.
  • If most players come from one country or age group, predictions might not generalize well.
  • Ages below 5 or above 100 might indicate incorrect or fabricated data.

Chosen Method: K-Nearest Neighbors (K-NN) Regression

I have chosen K-Nearest Neighbors (K-NN) regression to predict players' total playtime based on their registration information and gameplay styles. This method is simple and does not assume a specific relationship between the input features and the target variable.

Why is this method appropriate?

K-NN regression is suitable for our analysis because it predicts a player's total playtime by averaging the playtimes of the 'k' most similar players. This approach is intuitive and effective, especially when there is no clear assumption about the data's underlying distribution.

Assumptions of K-NN Regression

The main assumption of K-NN regression is that similar players have similar behaviors. This means players with comparable registration details and gameplay styles are likely to have similar total playtimes. Additionally, the algorithm assumes that the chosen distance metric accurately reflects the similarity between players.

Potential Limitations

One significant limitation of K-NN regression is its computational intensity. It requires calculating the distance between the target player and all players in the dataset, which can be time-consuming with large datasets. Moreover, the algorithm is sensitive to irrelevant or noisy features, which can distort distance calculations and negatively impact prediction accuracy. Choosing an appropriate value for 'k' is also crucial; a small 'k' may lead to overfitting, capturing noise in the data, while a large 'k' might oversmooth the predictions, missing important patterns.

Model Evaluation and Selection

To evaluate the performance of our K-NN regression model, we will use the Root Mean Squared Error (RMSE) metric, which measures the square root of the average squared differences between the observed and predicted playtimes. A lower RMSE indicates better predictive accuracy. We will also employ cross-validation techniques, such as k-fold cross-validation, to assess the model's generalization ability and to determine the optimal value of 'k' that minimizes the RMSE.

Data Preparation for Modeling

Proper data preparation is essential for the effectiveness of K-NN regression. We will begin with data cleaning to handle missing values and outliers, ensuring the quality of the dataset. Feature scaling is crucial, as K-NN is sensitive to the scale of input features; therefore, we will standardize the features to have zero mean and unit variance. Additionally, we will perform feature selection to retain only the most relevant features, reducing dimensionality and improving model performance. Finally, we will split the dataset into training and testing sets, typically using a 70-30 or 80-20 split, to evaluate the model's performance on unseen data and ensure its generalizability.
